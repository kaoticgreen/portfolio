Dialogue System Documentation

Project Overview

This is a technical documentation sample created for a large-scale RPG project. The goal was to provide a **clear, unambiguous specification** for the engineering team on how the in-game dialogue system should handle branching paths, conditional checks, and localized text variables.

Key Contributions

  • Authored 100% of the documentation, defining syntax and logic.
  • Created **UML diagrams** (or similar flowcharts) to visualize complex nested choices.
  • Established the **naming convention** for all dialogue assets and script IDs.
