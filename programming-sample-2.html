Simple AI Behavior Tree

Project Goal & Technical Stack

This project showcases a lightweight AI system designed for non-player characters (NPCs) and enemies. The system uses a simple **State Machine** pattern to manage behaviors like patrolling, seeking a target, and attacking, demonstrating clean transitions and robust logic.

Tools/Languages: Unity, C#, State Machine Pattern, Raycasting/Line-of-Sight Checks, and Git.

Technical Highlights

  • **Clear State Transitions:** Behavior is managed by a single `AI_Controller` script responsible for smooth transitions between `Patrol`, `Engage`, and `Rest` states.
  • **Line-of-Sight Logic:** Implemented efficient raycasting with layer masks to determine if a player is visible and within engagement range.
  • **Performance:** Designed to be highly performant by utilizing coroutines for non-critical tasks (like searching) and minimizing update logic.

Code Snippet: State Transition Logic

Below is a key code snippet showing how the AI decides to switch from Patrolling to Engaging:

// Excerpt from AI_Controller.cs
private void Update()
{
    switch (currentState)
    {
        case AIState.Patrol:
            HandlePatrol();
            if (CheckForTarget())
            {
                TransitionToState(AIState.Engage);
            }
            break;
        case AIState.Engage:
            HandleEngage();
            if (!IsTargetVisible())
            {
                TransitionToState(AIState.Rest);
            }
            break;
        // ... other cases
    }
}
