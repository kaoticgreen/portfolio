Custom Editor Tool

Project Goal & Technical Stack

This sample demonstrates a **custom editor window** built to simplify and accelerate the process of placing level data and assigning narrative triggers. The primary goal was to reduce the time designers spent performing repetitive, error-prone tasks in the standard inspector.

**Tools/Languages:** Unity Editor scripting, C#, Custom Inspectors, Editor GUILayout, and Scriptable Objects for configuration.

Technical Highlights

  • **Enhanced Workflow:** Consolidated six separate configuration steps into one streamlined editor window.
  • **Error Reduction:** Implemented input validation and automated path checking to prevent common designer errors (e.g., missing component references or incorrect file paths).
  • **Integration:** Utilized custom draw methods to integrate the tool seamlessly with existing project architecture, ensuring maintainability.

Code Snippet: Editor Window Setup

Below is a key code snippet showing the basic setup for a custom Editor window:

// Excerpt from LevelToolWindow.cs
public class LevelToolWindow : EditorWindow
{
    // Function to initialize and display the window
    [MenuItem("Tools/Robert/Level Placement Tool")]
    public static void ShowWindow()
    {
        GetWindow("Level Placer");
    }

    private void OnGUI()
    {
        GUILayout.Label ("Place New Scene Trigger", EditorStyles.boldLabel);
        
        // Example: Drawing a custom field for designer input
        triggerName = EditorGUILayout.TextField("Trigger Name:", triggerName);

        if (GUILayout.Button("Generate Trigger"))
        {
            // Logic to create and configure the GameObject
            GenerateTrigger();
        }
    }
}
