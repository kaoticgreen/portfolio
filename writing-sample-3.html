Quest Flowchart & Script

Project Overview

This sample demonstrates my ability to translate a narrative goal into a **structurally sound, implementable quest** that accounts for player choice and conditional branching. This document serves as the primary handover for scripters and level designers to build the quest logic.

Key Technical Components

  • **Flowchart Logic:** Utilized visual tools (e.g., Miro, Visio) to map out all possible quest states, failure points, and rewards.
  • **Scripting Conventions:** Adhered to strict formatting for dialogue delivery, including tags for required emotional tone, voice acting notes, and localization identifiers.
  • **Conditional Branching:** Integrated **programming logic checks** (e.g., `IF PlayerHasItem("MacGuffin")`) directly into the script for clear implementation requirements.

Excerpt: Scripted Dialogue

The following shows how dialogue is formatted for scripting, including internal tags:

// SCENE START: VillageElder_Entrance_01
NPC_Elder: [Tone: Anxious] Thank the heavens! The artifact—did you retrieve it?

// BRANCH 1: Player has the artifact (Condition: GlobalVar.Artifact_Obtained == TRUE)
PLAYER: I have it right here.
NPC_Elder: [Tone: Relief] Excellent. This saves us all. *QuestComplete(Quest_ArtifactRetrieval)*

// BRANCH 2: Player has failed (Condition: GlobalVar.Artifact_Destroyed == TRUE)
PLAYER: It was destroyed. I couldn't save it.
NPC_Elder: [Tone: Despair] Oh, the calamity! This changes everything. *QuestFail(Quest_ArtifactRetrieval)*

Conclusion

This sample highlights my commitment to **interdisciplinary documentation**. By clearly scripting narrative content with embedded technical logic, I minimize ambiguity and reduce friction during the implementation and QA phases.
