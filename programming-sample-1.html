Unreal Engine

Project Goal

The First Person Shooter tutorial was created to help developers implement a robust, networked third-person character controller and a flexible projectile firing system using C++ within Unreal Engine. The primary goal was to create a responsive player experience and a modular, easily expandable firing mechanic.

Tools/Languages: Unreal Engine, C++, and Blueprints

Technical Highlights

  • C++ Character Core: Built from the `ACharacter` base class, implementing C++ input bindings via `SetupPlayerInputComponent` for responsive, frame-independent player control.
  • Component-Driven Projectiles: Designed a reusable `AProjectile` actor class that leverages the `UProjectileMovementComponent` for robust, physics-driven movement, interpolation, and homing capabilities.
  • Dynamic & Decoupled Spawning: Implemented a `Fire` method on the character to dynamically spawn projectile actors using `UWorld::SpawnActor`, cleanly decoupling the character's logic from the projectile's behavior.
  • Collision & Events: Utilized a `USphereComponent` on the projectile for efficient collision detection and bound a delegate to its `OnHit` event to handle impact logic (e.g., applying damage, playing effects).

Code Snippet: Character Input Binding (C++)

This snippet from the `AMyCharacter` class shows how player input is bound directly in C++ for maximum performance and clarity.

// In AMyCharacter::SetupPlayerInputComponent
void AMyCharacter::SetupPlayerInputComponent(UInputComponent* PlayerInputComponent)
{
    Super::SetupPlayerInputComponent(PlayerInputComponent);

    // Ensure the component is valid
    check(PlayerInputComponent);

    // Bind movement axes
    PlayerInputComponent->BindAxis("MoveForward", this, &AMyCharacter::MoveForward);
    PlayerInputComponent->BindAxis("MoveRight", this, &AMyCharacter::MoveRight);

    // Bind look axes
    PlayerInputComponent->BindAxis("Turn", this, &APawn::AddControllerYawInput);
    PlayerInputComponent->BindAxis("LookUp", this, &APawn::AddControllerPitchInput);

    // Bind actions
    PlayerInputComponent->BindAction("Jump", IE_Pressed, this, &ACharacter::Jump);
    PlayerInputComponent->BindAction("Fire", IE_Pressed, this, &AMyCharacter::Fire);
}

Code Snippet: Projectile Header (C++)

The header for the `AMyProjectile` class, demonstrating the component-based design.

#pragma once

#include "CoreMinimal.h"
#include "GameFramework/Actor.h"
#include "MyProjectile.generated.h"

class USphereComponent;
class UProjectileMovementComponent;

UCLASS()
class AMyProjectile : public AActor
{
    GENERATED_BODY()

public:
    AMyProjectile();

protected:
    // Collision component
    UPROPERTY(VisibleDefaultsOnly, Category = "Projectile")
    USphereComponent* CollisionComponent;

    // Movement component
    UPROPERTY(VisibleAnywhere, Category = "Movement")
    UProjectileMovementComponent* ProjectileMovementComponent;

    // Function called when projectile hits something
    UFUNCTION()
    void OnHit(UPrimitiveComponent* HitComponent, AActor* OtherActor, UPrimitiveComponent* OtherComp, FVector NormalImpulse, const FHitResult& Hit);
};

Conclusion & Repository

This project demonstrates a strong grasp of core Unreal Engine C++ principles, including component-based design, event handling with delegates, and the proper separation of concerns between different gameplay actors.
