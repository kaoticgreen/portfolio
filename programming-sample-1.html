Inventory Management System

Project Goal & Technical Stack

This system was developed to provide a **flexible, object-oriented solution** for handling item data, storage, and transfers within a Unity project. The core design principles were **modularity** and **extensibility**.

Tools/Languages: Unity, C#, Scriptable Objects, Interfaces, and Git.

Technical Highlights

  • **Data-Driven Design:** Utilizes **Scriptable Objects** to define item data (stats, sprites, descriptions) outside of runtime components.
  • **Modular Inventory:** Implements an `IInventory` interface, allowing the same logic to be used for player bags, chests, and vendor shops.
  • **Event System:** Uses C# events/delegates to decouple the inventory backend from the UI display, ensuring high performance.

Code Snippet: Inventory Interface

Below is a key code snippet demonstrating the use of an interface for a robust, reusable component:

public interface IInventory
{
    int SlotCount { get; }
    void AddItem(ItemData item, int quantity);
    bool RemoveItem(ItemData item, int quantity);
    bool ContainsItem(ItemData item, int quantity);
    // ... other core inventory methods
}

This project demonstrates strong C# principles, including adherence to SOLID design patterns and proper separation of concerns. The full source code and documentation are available on my GitHub repository.
